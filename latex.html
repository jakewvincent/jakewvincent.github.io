TeX editors

As Windows user, I have had a hard time finding a TeX editor that doesn't feel outdated or clunky. MacOS users have had TeXShop available for some time, which always seemed to me to be an excellent editor-previewer that's both cleaner and more functional than the standard Windows editor-previewers that I tried (TeXworks, TeXstudio). There are better options available for Windows users, but they weren't the first options I came across when I was starting out.

In my opinion, the best TeX editors are source code editors (text editors optimized for writing code) outfitted with extensions for compiling LaTeX documents. These are only slightly more complicated to set up, but in my experience they offer some advantages over the dedicated TeX editors.

  • No specialized TeX editor required (since you probably need a text editor anyway)
  • Suggestions/completions for in-text citations
  • More customizable builders/toolchains
  • All the advantages of using a source code editor (themes, extensions, etc.)

How to get it:

  1. Download and install VS Code: code.visualstudio.com
  2. Get the LaTeX Workshop extension:
    1. In VS Code, press Ctrl+Shift+X (Windows/Linux) or Shift+Cmd+X (Mac) to open the extensions panel
    2. Enter "latex workshop" into the search bar
    3. Click "LaTeX Workshop" to open the extension's page within the editor
    4. Click "Install"
    5. See instructions for setting up LaTeX Workshop: marketplace.visualstudio.com/latex-workshop
Advantages Disadvantages
  • Free
  • Open source
  • Cross-platform
  • Great syntax highlighting for TeX editing
  • PDF previewer opens within the editor (as a separate pane)
  • Customizable builders/toolchains
  • Works well out of the box (as long as you already have a TeX distribution)
  • Not as lightweight as Sublime Text 3
  • Changing LaTeX builders/toolchains (in LaTeX Workshop) is not as streamlined as in LaTeXTools for Sublime

How to get it:

  1. Download Sublime Text 3: sublimetext.com/3
  2. Install Package Control for Sublime: packagecontrol.io
  3. Get the LaTeXTools package:
    1. In Sublime (after having installed Package Control), press Ctrl+Shift+P (Windows/Linux) or Cmd+Shift+P (Mac)
    2. Start typing "install"
    3. Click "Package Control: Install Package"
    4. Start typing "latextools"
    5. Click "LaTeXTools"
    6. See instructions for setting up LaTeXTools: packagecontrol.io/packages/LaTeXTools
Advantages Disadvantages
  • Very lightweight editor
  • Cross-platform
  • Customizable builders/toolchains
  • Not free (unless you're okay with occasional reminders to pay)
  • Closed-source
  • Separate PDF viewer required

Reference management

When I first started using BibTeX with LaTeX, I would manually create a .bib file per-document with reference information copy-pasted from Google Scholar or wherever. There are much more efficient ways to do this. My favorite way is to use reference management software that keeps track of your library (and organizes it for you) and generates a single .bib file so that your references are on tap whenever you need them. When paired with a TeX editor with suggestions/completions culled from your .bib file (like either of those mentioned above), this makes for practically effortless reference management. See how the system I use works below.

Mendeley
I use the Mendeley desktop application to help keep track of all of the papers I've read (or—let's be honest—thought about reading) for my research. Papers can be drag-and-dropped into the application, or downloaded into a folder that Mendeley watches, and after doing either of those things, the application saves the pdf to my library and gives it an automatically generated, easy-to-read filename. The application is also a pdf viewer and annotator. In the settings, you can ask Mendeley to manage a bib file in a directory of your choosing. Do this, and any little change made in the pdf information panel is immediately reflected in the bib file.

Referencing the bib file in the LaTeX document
With a bib file in a particular location, it's easy to just always reference its full path in your tex document. If my bib file is called "my_library.bib" and is located in my documents, I would call this command right where I want my references printed if I'm using BibTeX to generate my references:

\bibliography{C:/Users/username/Documents/my_library}

And if I'm using Biber, I would call this command somewhere in my preamble:

\addbibresource{C:/Users/username/Documents/my_library.bib}

I like to create a shortcut to automatically fill in the basic structure of my tex documents (including calling all the packages I typically use and the command with the path for my bib file already filled in). In both of the text editors mentioned above, this can be done by creating a "snippet" (a file where your template is stored) that can be summoned by typing a particular string and hitting tab or enter. Once you save this as a .tex file, if you're using one of the above editors, you'll get suggestions/completions for your bibliography entries every time you start the first part of a citation command, as shown below. This way you don't have to memorize citation keys; you can start typing any information connected to the particular reference you're looking for, as long as that information is in that reference's entry in your bib file.
